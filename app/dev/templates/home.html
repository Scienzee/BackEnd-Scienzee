Valição e geração de Token
POST /api/token/

Essa consulta verifica as credencias fornecidas pelo usuário e retorna 02 (dois) tokes de sessão caso seja verificada a correlação e existência do usuário e senha.
O refresh token é um retorno padrão do djangorestframework-simplejwt validade default de 24h. Esse token é base para consulta que valida seu uso por 24h.
O access token é um retorno padrão do djangorestframework-simplejwt validade default de 5min. Pode ser usado diretamente nas requisições sem a necessidade validação

Requisição

                    
                        Headers: Content-Type: application/json
                        body{
                            "email": "usuario@scienzee.com.br",
	                        "password": "senha"
                        }
                        Status Code: 200

Response (Ok)

                    
                        {
                            "refresh": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJ0b2tlbl9[...]UzKtZohsr0",
                            "access": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJ0b2t[...]zMDMyMDcyY2U1NDQ2N8rxu18Fhk"
                        }
                        Status Code: 200

Response (Credencias inválidas)

                    
                        {
                            "detail": "No active account found with the given credentials"
                        }
                        Status Code: 401

GET /api/staff?name=NomeConsulta&token=SeuToken

Essa consulta retorna um array de dicionários com base no parâmetro de busca "nome do policial", ou seja, é um modelo de consulta parcial (LIKE).
Obs: O parâmetro de consulta deve ser passado sem chaves.*

Query Params
KEY VALUE
nome parametro_de_consulta
token seu_token

Response (Ok)

        
            [{
                "nome_completo": "nome completo do miltar",
                "nome_guerra": "nome de guerra do militar",
                "rgpm": "***0000-*",
                "situacao": "situacao do militar",
                "imagem": url_da_imagem_do_militar
            }]

Response (Error)

Caso a API não reconheca token o retono da request será: 
        
            {
                'Error': 'Unauthorized'
            }

Status Code: 401
Caso o método HTTP seja diferente de GET o retono da request será: 
        
            {
                'Error': 'Method Not Allowed'
            }

Status Code: 405
Caso a consulta não encontre nenhum registro para o parâmetro fornecido o retono da request será: 
        
            {
                'Error': 'Not Acceptable'
            }

Status Code: 406

GET api/staff/{email}/full?token=SeuToken

Essa consulta retorna um dicionário com base no parâmetro de busca e-mail, ou seja, é um modelo de consulta integral.
Obs: O parâmetro de consulta deve ser passado sem chaves.*

Query Params
KEY VALUE
token seu_token

Response (Ok)

        
            {
                "email": "nome.sobrenome@ac.gov.br",
                "imagem": "http://url_imagens_policial_png",
                "ind_id": 0000,
                "nome_completo": "NOME COMPLETO POLICIAL",
                "nome_guerra": "NOME DE GUERRA DO POLICIAL",
                "pg_id": 00,
                "posto_graduacao": "CB",
                "rgpm": "***0000-*",
                "sit_id": 00,
                "situacao": "ATIVA",
                "uni_id": 00,
                "unidade": "UNIDADE DE LOTACAO"
            }

Os campos ind_id, pg_id, sit_id e uni_id representam respectivamente o id do indivíduo, id do posto ou graduação, id da situação e id da unidade.

Caso a API não reconheca token o retono da request será: 
        
            {

            {
                'Error': 'Method Not Allowed'
            }

Caso a consulta não encontre nenhum registro para o parâmetro fornecido o retono da request será:
